Shepherding Algorithms


Links

:Github

: Website

Overview

This project was done as a part of my masters thesis, the main aim of which was to create a series of algorithms that can be used by a UAV/multiple UAV's to autonomously herd farm animals. This was accomplised by creating an Agent-Based Modelling (ABM) simulation. ABM is a form of computational modelling which allows for the study of interactions between individual, autonomous agents and with their environment. Agents are essentially objects which are given a number of behaviours/actions to simulate some real-world entity, while the environment is area they enclosed in. The simulation was created using object-oriented JavaScript and p5.js, a library based on the processing programming language which contains a number of functions and for working with a canvas and vectors.

Animal Agents

Animal models were creating using Craig Reynolds Flocking Algorithm, Craig Reynolds Steering Behaviour Algorithms and a number of addtional behaviours to simulate herd animal behaviour. Reynolds Flocking model emulates the flocking or herding behavior of animals by giving them three behaviors. These behaviours are seperation(where animal agents steer to avoid collision with each other), alignment (where animal agents steer to the average velocity and heading of their neighbours) and cohesion (where animal agents move the the average position of its neighbours). Once applied to agents in a simulation, these agents will inherently display the emergent behaviour displayed by animals when flocking, herding or schooling.

Steering for each agent is calculated using Reynolds steering alogrithm where
steering = desired velocity - velocity.
Additional behaviors added to animal agents to have them emulate herd animal behaviour where uav separation (where animals steer away from a UAV), goal setting (where animals steer toward a goal point), novelty separation (where animals agents will steer to avoid novel or hazardous environments) and wander(when not in the presence of a UAV animal agents will move slowly in random directions to emulate grazing).

Each animal agent has a neighbourhood which represents the extent of an animal agents sensory range.The velocity of each animal and the force strength of each behaviour mentioned above changes depending on the presense of other animal agents, uav agents or environmental factors in this neighbourhood.

UAV Agents

The aim of the UAV's in this simulation is to form a group of animal agents and move that group to some goal point. A UAV's steering is also calculated Reynolds Steering algoritms. Two different methods were implemeted for a UAV agent to locate animal agents, these are named gps identification and oracle identification. When using gps identification animal agent locations are passed directly to a UAV agent. When using oracle identifiaction, a seperate UAV is introduced (named the oracle) which collects the location of animals agents and passes them to the UAV agent.

A UAV moves towards various steering points to accomplish its goals. These steering points are dependant on its own position, positions of the animal agents, velocity of the animal agents, if the animal agents are in the vicinity of a harmful area and the position of the goal/exit point.

Facial Recognition Using a UAV


Links

:Github

Overview

The aim of this project was to give an AR Drone 2.0 the ability to fly to a specific person and once it was within a certain distance to that person, fly away again. This was accomplised through the use of facail recognition. This project was completed using NodeJS and a number of libraries. The libraries utilised for this project included express, socketIO, opencv4nodejs and ar-drone.

Facail Recognition

OpenCV and it's built in LBPH (Linear Binary Pattern Histogram) algortithm was utilied for computer vision functionalities. To train the facail recognition classifier, a pre-built facail detection classifier was used to decect a specific face. 100 images of a face were collected. These images were broken into histograms which are used for comparison when trying to detect a specific face. When detecting a face the UAV flies of the ground and rotates, when a face is detected it broken into a histogram and compared with the histogram created during the training period. Once compared a confidence value is outputted. If the confidence value is lower than a certain percentage, the UAV will continue to rotate in search of the correct person. If the confidence value is higher than a certain percentage, the UAV will move toward that person. Once the UAV is at a certain distance to that person, the UAV will fly backwards and land.
